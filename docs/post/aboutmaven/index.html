About Maven

Apache Maven is a software project management and comprehension tool. Based on the concept of a project object model (POM), Maven can manage a project’s build, reporting and documentation from a central piece of information.

Maven是一个Java项目的管理和构建工具

  • Maven使用pom.xml定义项目内容，并使用预设的目录结构
  • 在Maven中声明一个依赖项可以自动下载并导入classpath
  • Maven使用groupId，artifactId和version唯一定位一个依赖

Maven主要功能

  • 提供了一套标准化的项目结构
  • 提供了一套标准化的构建流程（编译，测试，打包，发布など…）
  • 提供了一套依赖管理机制

项目结构

a-maven-project
├── pom.xml   <-- 描述文件
├── src  <--入口
│   ├── main
│   │   ├── java <-- java code 
│   │   └── resources <-- 资源文件位置
│   └── test <-- 测试用源码
│       ├── java <-- java code
│       └── resources  <-- 资源文件位置
└── target <--打包生成文件存放

pom.xml

<project ...>
	<modelVersion>4.0.0</modelVersion>
	<groupId>com.wintercat.jp</groupId>
	<artifactId>hello</artifactId>
	<version>1.0</version>
	<packaging>jar</packaging>
	<properties>
        ...
	</properties>
	<dependencies>
        <dependency>
            <groupId>commons-logging</groupId>
            <artifactId>commons-logging</artifactId>
            <version>1.2</version>
        </dependency>
	</dependencies>
</project>

依赖声明

<dependency>
    <groupId>commons-logging</groupId>  <!-- Java的包名 -->
    <artifactId>commons-logging</artifactId>  <!--Java的类名-->
    <version>1.2</version>  <!--版本-->
</dependency>

依赖关系

Maven定义了几种依赖关系，分别是compile、test、runtime和provided

scope 説明
compile 编译时需要用到该jar包
test 编译Test时需要用到该jar包
runtime 编译时不需要，但运行时需要用到
provided 编译时需要用到，但运行时由JDK或某个服务器提供

pom.xml

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.3.2</version>
    <scope>test</scope> <!--依赖关系-->
</dependency>
